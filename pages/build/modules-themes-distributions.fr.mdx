# Projets Drupal

Il existe une multitude de projets Drupal, voici une courte liste de modules et de thèmes stables qui peuvent être utilisés dans des cas d'utilisation courants, ainsi que quelques distributions. La plupart des modules peuvent interagir ensemble.

D'autres projets peuvent être découverts via l'initiative [Project Browser](https://www.drupal.org/project/project_browser).

## Modules

Les modules fournissent généralement une fonctionnalité spécifique. Ils peuvent fonctionner seuls ou en collaboration avec d'autres modules.

### Expérience de développement

| Nom                                                                     | Description             |
|--------------------------------------------------------------------------|-------------------------|
| [Devel](https://www.drupal.org/project/devel)                            | Améliore la plupart des tâches de développement, les expose sous forme de commandes Drush et améliore le débogage des variables avec des bibliothèques telles que Kint. |
| [Module Builder](https://www.drupal.org/project/module_builder)          | Génère une structure ou un "squelette" pour un module, ainsi que des indications sur la façon de les compléter. Utile pour les développeurs débutants pour apprendre comment fonctionne le code Drupal, et pour les développeurs chevronnés trop paresseux pour rechercher quels arguments une fonction doit prendre. |
| [Stage File Proxy](https://www.drupal.org/project/stage_file_proxy)      | Obtenez les fichiers de production sur un serveur de développement à la demande plutôt que de tout synchroniser avec rsync.             |
| [Web Profiler](https://www.drupal.org/project/webprofiler)               | Ajoute une barre d'outils en bas de chaque page et affiche toutes sortes de statistiques telles que le nombre de requêtes de base de données chargées sur la page, les services utilisés, et bien plus encore. |
| [Backup and Migrate](https://www.drupal.org/project/backup_migrate)      | Sauvegarde et restaure votre site Drupal. |
| [Masquerade](https://www.drupal.org/project/masquerade)                  | Permet aux utilisateurs avec suffisament de permissions de basculer de compte utilisateur sans nécessité de se ré-identifier. | 
| [Config Pages](https://www.drupal.org/project/config_pages)              | Vous n'avez plus besoin de coder pour obtenir des formulaires de configuration pour votre module ou toute autre configuration personnalisée. |
| [Config Ignore](https://www.drupal.org/project/config_ignore)            | Ignore certains éléments de configuration lors de l'importation. Typiquement lorsque la configuration doit être considérée comme du contenu et non comme faisant partie du référentiel. |
| [Examples for developers](https://www.drupal.org/project/examples)       | Exemples pour les blocs, le cache, la configuration des entités, la tâche cron, l'API de base de données, les e-mails, les événements, l'API de formulaire, les champs, les autorisations de champ, les fichiers, les hooks, JavaScript, les types de contenu, les pages, la pagination, PHPUnit, les types de plugin, la file d'attente, les tests simples, le wrapper de flux, le tri des tables, les tests, la visite guidée. |
| [Reroute Email](https://www.drupal.org/project/reroute_email)            | Intercepte tous les e-mails sortants d'un site Drupal et les redirige vers une adresse e-mail prédéfinie configurable. |
| [Event Condition Action](https://www.drupal.org/project/eca)             | Crée des règles qui sont déclenchées par des événements. Quelques [excellentes vidéos tutorielles](https://tube.tchncs.de/c/eca/videos?s=1) des mainteneurs. |

### Expérience éditoriale

#### Solutions éditoriales

**Mise en page**

| Nom                                                                     | Description             |
|--------------------------------------------------------------------------|-------------------------|
| [Layout Builder](https://www.drupal.org/docs/8/core/modules/layout-builder) (core) | Permet aux rédacteurs de contenu et aux constructeurs de site de créer facilement et rapidement des mises en page visuelles pour l'affichage du contenu |
| [Paragraphs](https://www.drupal.org/project/paragraphs)                  | Au lieu de mettre tout leur contenu dans un champ de corps WYSIWYG comprenant des images et des vidéos, les utilisateurs finaux peuvent maintenant choisir dynamiquement parmi des types de paragraphes prédéfinis indépendants les uns des autres |
| [Gutenberg](https://www.drupal.org/project/gutenberg)                    | Apporte les puissantes fonctionnalités d'administration de l'éditeur Gutenberg |

Gutenberg et Layout Builder bénéficient d'un soutien croissant au sein de la communauté.
- Il y a un projet pour [découpler Layout Builder avec React](https://www.previousnext.com.au/blog/what-if-pitching-decoupled-layout-builder)
- L'intégration de Gutenberg avec Drupal est bien maintenue et a récemment reçu [un financement pour son développement](https://wptavern.com/automattic-donates-e20000-to-fund-next-phase-of-drupal-gutenberg-development)

**Flux de travail / Modération de contenu (core)**

> Le module [Content Moderation](https://www.drupal.org/docs/8/core/modules/content-moderation/overview) vous permet d'étendre les états "non publié" et "publié" de Drupal pour le contenu. Il vous permet d'avoir une version publiée qui est en ligne, mais de disposer d'une copie de travail distincte qui est en cours de révision avant sa publication. Cela est réalisé en utilisant des flux de travail pour appliquer différents états et transitions aux entités selon les besoins.

### Traduction

Drupal est entièrement multilingue dès le départ, il suffit d'activer quelques modules et de les configurer.
Ensuite, tout ce dont vous avez besoin est d'ajouter des langues.

Voici quelques modules multilingues qui améliorent l'expérience de traduction.

| Nom                                                                     | Description             |
|--------------------------------------------------------------------------|-------------------------|
| [Translation Management Tool](https://www.drupal.org/project/tmgmt)      | Fournit un ensemble d'outils pour traduire du contenu à partir de différentes sources. La traduction peut être réalisée par des personnes ou des services de traduction de toutes sortes. Il s'appuie sur les outils et structures de langue existants dans Drupal et peut être utilisé dans des scénarios de flux de travail automatisés. |
| [TMGMT DeepL Integration](https://www.drupal.org/project/tmgmt_deepl)    | Intègre l'API DeepL avec le module Translation Management Tools (TMGMT). |

## Thèmes

Les thèmes vous permettent de modifier l'apparence et l'expérience utilisateur du frontend ou du backend, et peuvent être utilisés tels quels ou par héritage.

### Frontend

**Olivero** (core) est le thème frontend par défaut pour Drupal. Il s'agit d'un thème moderne axé sur l'accessibilité et l'utilisabilité.

### Backend

**Claro** (core) est le thème backend par défaut. Il peut être étendu par le [Gin Admin Theme](https://www.drupal.org/project/gin).

![Gin Admin Theme](/assets/images/docs/gin-drupal-admin-theme.webp)

> Une nouvelle interface utilisateur radicalement différente associée à des fonctionnalités telles que le mode sombre donnera un coup de jeune à l'interface d'administration de votre Drupal. Le thème Gin inclut également des éléments qui sont actuellement hors du champ d'application de Claro et/ou des personnalisations que nous expérimentons pour l'avenir. Construit sur la base de Claro par l'un des principaux concepteurs de Claro et du Design System Drupal.

[Rechercher des thèmes Drupal](https://www.drupal.org/project/project_theme)
